# .(dot)
- 현재 폴더

# new
- class 타입의 instance를 생성해주는 역할
- new 연산자를 통해 Heap영역에 데이터를 저장할 공간을 할당 받고 reference value(참조값)을 객체에게 반환하여 주고 이어서 생성자를 호출함

```
  A	   a	    =			new				    A();
자료형 참조값저장	Heap할당,인스턴스생성,참조값 a에게 리턴		생성자 호출
```

# Call by 간단정리

### value
- 장점 : 복사하여 처리, 안전함, 원래값이 보존
- 단점 : 복사하여서 메모리 사용량 증가

### reference
- 장점 : 복사하지 않고 직접 참조를 하여 빠름
- 단점 : 직접 참조를 하기에 원래값이 영향을 받음(리스크)

# OOP
- 세상에 있는 것을 객체화 하여 프로그래밍 하는 방법론

## Object
- 인지할 수 있는 모든 것
- S/W세계에 구현할 대상

## Abstraction
- 문제에 대하여 의존적이다
- 공통적인 상태, 행동을 묶어서 이름을 붙인다

## Class
- 추상화를 통하여 상태와 행위를 정의하여 여러가지 인스턴스를 생성한다.

## Instance
- class를 바탕으로 S/W 세계에 구현된 구체적인 실체
	- 객체를 소프트웨어에 실체화 하면 그것을 'instance'라 한다.
	- 실체화된 instance는 메모리에 할당된다.
- instance라는 용어는 반드시 클래스와 객체 사이의 관게로 한정지을 필요는 없다


### Object vs Instance
- 객체는 현실세계에 가깝고, 인스턴스는 소프트웨어세계에 가깝다.
- 객체는 '실체', 인스턴스는 '관계'에 초점을 맞춘다
